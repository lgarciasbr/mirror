# v0.10.12 — Pending Release Notes Ref Refresh

**Date:** 2026-05-25

## Highlights

- `runtime release-notes pending` now safely fetches `remote/branch` refs before
  reading release notes.
- Added `--no-fetch` for local smoke tests such as `--ref HEAD --no-fetch`.
- Welcome release awareness now matches short remote commit ids against full tag
  hashes.
- Updated runtime help and release-note documentation for the new pending-notes
  fetch behavior.

## Why This Patch Exists

Production validation of the v0.10.11 stub showed that the welcome and prompt
routing were working: opening Mirror detected a new stable release and the user
question triggered `runtime release-notes pending`.

But the pending command returned `Pending releases: none`. The reason was that
welcome had detected the new remote stable commit without fetching, while
`release-notes pending` read the local `origin/stable` ref. If that local ref was
stale, the command could not see the newly published release note.

## What Changed

When the pending release-note source looks like a remote-tracking ref such as
`origin/stable`, the command now runs a safe fetch for that branch before reading
notes:

```bash
git fetch origin stable
```

This updates remote-tracking refs only. It does not merge, checkout, migrate,
modify the working tree, or update the runtime.

For local release-candidate smoke tests, use:

```bash
uv run python -m memory runtime release-notes pending --ref HEAD --no-fetch
```
