# v0.10.15 — Update Action Routing

**Date:** 2026-05-25

## Highlights

- Added an `mm-update` skill for natural-language Mirror runtime update requests.
- Routes `update my Mirror` to `uv run python -m memory runtime update`.
- Documents the command in help, AGENTS/CLAUDE context, and the command reference.
- Explicitly prevents agents from substituting manual `git pull`, `git fetch`, or
  migration commands for the safe updater.

## Why This Patch Exists

Production validation showed that the welcome action `update my Mirror` was still
ambiguous to the agent. Instead of running the safe updater, it asked whether the
user meant a code update or an identity/memory update.

For the welcome to behave like product UX, both suggested actions must be
routable. Release-note questions already route to cumulative pending notes; the
update action now routes to the runtime updater.

## What Changed

A new skill, `mm-update`, handles natural-language variants such as:

```text
update my Mirror
apply the Mirror update
install the new Mirror version
```

The skill runs:

```bash
uv run python -m memory runtime update
```

and shows the output verbatim.
