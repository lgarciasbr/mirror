# v0.11.0 — Read-only Web Refinement

## Digest

This release closes CV13.E1, the first Mirror Web Experience epic. It refines
the local read-only web surface after the 1.0 visibility foundation: Workspace
becomes the front door, Identity becomes easier to scan, memory categories and
search become navigable, and conversation cards become more useful without
introducing writes or background operations.

## Highlights

- Workspace opens first and the main navigation is ordered Workspace, Identity,
  Docs.
- The header now presents Mirror Mind as the app identity and the current Mirror
  user as a separate avatar/name lockup.
- Docs, Workspace, Identity, memory category pages, and search results now share
  the same contextual-bar rhythm.
- Identity persona tokens use stable three-letter icons.
- Identity memory categories open read-only category pages backed by surface
  DTOs.
- Header search opens a read-only results page over recent retained memories.
- Workspace conversation cards show clearer message, persona, journey, and date
  metadata.

## Narrative context

CV9.E6 made the Mirror visible on the web. CV13.E1 makes that visibility feel
less like a diagnostic surface and more like a product. The work stayed inside a
read-only boundary, but it changed the user's path through the app: Workspace is
now the natural home, Identity remains a reflective map, Docs sits inside the
same visual grammar, and memory/conversation surfaces have enough affordance to
invite exploration.

This is a pre-1.0 minor release. The earlier CV13 planning label was `v1.1`, but
Mirror Mind has not yet crossed the CV9 `v1.0.0` boundary. Under the prospective
versioning rule, closing this epic increments the current minor version to
`v0.11.0`.

## What changed

- Added CV13.E1 roadmap stories and marked S1–S5 done.
- Made Workspace the default perspective when no stored preference exists.
- Removed task-oriented noise from Workspace metrics and tabs.
- Removed the redundant perspective badge from the header.
- Reworked the header and perspective switcher visual treatment.
- Moved Docs navigation into the main content frame so Docs aligns with the
  other perspectives.
- Generated three-letter persona tokens in the Atlas surface read model.
- Added a memory category drilldown surface and web route.
- Replaced misleading progress bars with low/medium/high presence chips.
- Added lexical read-only search over recent memory summaries.
- Added a specialized renderer for Workspace conversation cards.

## Conscious exclusions

- No writes or editing flows.
- No multi-Mirror selector or profile/preferences page.
- No Mirror or journey configuration console.
- No conversation transcript page.
- No conversation retitle operation.
- No semantic/vector web search.
- No LLM calls during web page rendering.
- No background Pi agent execution.
