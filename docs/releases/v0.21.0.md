# v0.21.0 — Scene Workspace Home

**Date:** 2026-06-02

## Highlights

- Workspace now opens to a global **Current Scene** instead of automatically selecting the first journey.
- **Your Moment** separates global surfaces from **Your Journeys**: Current Scene, Conversations, and All journeys are no longer mixed into the journey tree.
- Scene Orientation is persisted, structured, and grounded in a bounded deterministic Scene read model.
- Global orientation can generate automatically only the first time; focused journey orientations are always user-invoked.
- Focused journey orientation appears as the first journey tab, before Briefing, Attachments, Conversations, and Settings.
- Conversations now render as a compact, date-grouped list suitable for many conversations, with journey shown as contextual metadata.
- All journeys now provides broader journey-field cards with status icons for parent and child journeys.

## Where We Started

CV15.DS1 gave Mirror Mind one-level journey hierarchy. That made the Workspace easier to organize, but it still behaved like a project browser: opening the Workspace selected a journey immediately, global context was ambiguous, and maintenance surfaces such as unassigned conversations leaked into the journey tree.

DS2 started from the question behind Cognitive Location: when the user opens Workspace, Mirror should answer where the user is situated now, not only list containers.

## What Changed

Workspace now carries a deterministic `scene` read model composed from the journey field, selected journey state, hierarchy, recent conversations, memories, decisions, open tasks, nearby journeys, and horizon signals. The read model is bounded before it reaches the LLM layer, and Scene remains a surface rather than a user-managed object.

The web console now distinguishes **Your Moment** from **Your Journeys**. Your Moment contains Current Scene, Conversations, and All journeys. Your Journeys contains only real journey navigation. Conversations and All journeys render inside the Workspace shell, so the sidebar remains present and navigation no longer breaks when moving between global Workspace surfaces.

Scene Orientation became a lifecycle rather than a raw model response. The orientation is stored internally under `scene_orientation`, normalized into title, summary, signals, and next movement, and marked outdated when the underlying Scene source hash changes. The UI renders outdated orientation with a subtle visual indicator and lets the user refresh it deliberately.

The invocation policy is intentionally asymmetric. Global Current Scene may generate automatically once when no orientation exists, giving the Workspace a living first impression. After that, updates are explicit. Focused journey orientations are always invoked by the user, because asking Mirror to read a specific journey should remain a deliberate gesture.

## Conscious Exclusions

- No automatic context inheritance between parent and child journeys.
- No memory, search, routing, Builder, extraction, task, or conversation-assignment merging across hierarchy.
- No user-managed Scene entity.
- No hidden regeneration loop after the first missing global orientation.
- No automatic mutation from orientation synthesis.
- No recursive hierarchy beyond the one-level foundation from v0.20.0.

## What We Learned

Cognitive location needs editorial restraint. Exposing every deterministic signal in the main card made the Scene feel like diagnostics instead of orientation. The better shape is layered: the read model stays explicit and bounded in the system, while the primary UI offers a readable orientation and keeps operational surfaces nearby.

We also learned that invocation matters. Orientation has interpretive weight, so generation should feel invited. The global first-run exception helps the Workspace come alive without turning every journey read into an unsolicited interpretation.

## Next Horizon

CV15 now has both layers: hierarchical organization and Scene-based cognitive location. The visible horizon is validation in daily use: how often orientations become stale, whether the structured synthesis stays useful, and whether the Workspace needs deeper search/filter controls for high-volume Conversations.
